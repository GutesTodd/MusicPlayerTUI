# Yandex Music CLI

> ⚠️ **Проект находится в активной разработке**

Современный терминальный клиент для Яндекс Музыки, построенный на клиент-серверной архитектуре.

## 🏗 Архитектура

Проект разделен на два основных компонента, взаимодействующих через JSON-RPC подобный протокол поверх TCP/Unix сокетов:

- **Backend**: Написан на Python с использованием `asyncio`. Отвечает за взаимодействие с API Яндекс Музыки (через `yandex-music-python`), управление очередью и воспроизведение звука через библиотеку `mpv`.
- **UI (TUI)**: Построен на фреймворке `Textual`. Предоставляет отзывчивый и визуально приятный интерфейс в терминале.

### Основные технологии:
- **Python 3.14+**: Использование последних возможностей языка.
- **Textual**: Мощный TUI фреймворк для создания интерфейса.
- **Dishka**: Инверсия зависимостей (Dependency Injection).
- **MPV**: Надежный движок для воспроизведения аудио.
- **Pydantic**: Валидация данных и схем общения.

## 🚀 Запуск

Для работы проекта необходимо наличие установленного в системе `mpv`.

### Установка зависимостей (через uv):
```bash
uv sync
```

### Запуск приложения:
```bash
python launcher.py
```
`launcher.py` автоматически запустит фоновый процесс бэкенда и откроет графический интерфейс.

## 🗺 План разработки (Roadmap)

Ниже представлен чек-лист задач по развитию модуля Яндекс Музыки:

- [ ] **Очередь треков**: Полноценное управление очередью с дополнительным окном просмотра.
- [ ] **Режим повтора**: Поддержка цикличного воспроизведения треков.
- [ ] **Интерактивные элементы**: Замена статических прогресс-баров на интерактивные слайдеры для перемотки трека и изменения громкости.
- [ ] **Кроссплатформенность**: Оптимизация сетевого взаимодействия через `asyncio.open_connection` для стабильной работы на Windows и macOS.
- [ ] **Медиатека**: Интеграция плейлиста «Мне нравится».
- [ ] **Рекомендации**: Реализация поддержки бесконечного потока «Моя волна».
